Linux 集群配置
集群是个协同工作的服务器，各有分工，对外表现个整体。 集群分类： 负载均衡型(LBC):分担服务器压力，每个服务器运算都分摊，提升工作效率， 高可靠用型(HAC)：保持服务状态的可用性。 高性能运算集群(HPC):提供单台服务器运算不了的性能 (一台服务器一万五左右1U服务器， 网页支持并发10w/小时。在线游戏棋牌游戏2k-3k/小时。) 负载均衡模式： 硬件负载均衡基于数据联路协议。 LVS的负载均衡是基于传输层协议。LVS负载均衡是把客户端访问目标地址修改成LVS之后的服务器的地址。把服务器响应的给客户端的源地址修改成LVS的地址，只负责地址转发。只建立完整的一次链接。数据包不会处理。支持B/S，C/S。 Nginx的负载均衡是基于应用层协议。Nginx负载均衡是客户端和Nginx建立一次链接，Nginx和后台服务器又要建立新的链接。建立两侧链接。数据包会处理。只支持B/S。 NAT模式： LVS必须是Linux服务器，集群在同一个网段中，真实服务器必须通过LVS与客户端连接。LVS有两个IP，一个IP连接客户端，一个IP与服务器连接。(NAT模式可以转换目标地址和端口，这叫端口映射。DNAT 目标地址转换。SNAT 源地址转 TUN模式： 客户端，LVS，服务器节点都在不同地点公网上。基于数据包地址二次封装。不支持端口映射。LVS必须是Linux。真实服务器随意。 DR模式： 集群在同一个广播域，LVS和真实服务器通过交换机相连。真实服务器多加个隐藏IP。客户端请求时候经过LVS。真实服务器直接响应客户端，不经过LVS服务器。不支持端口映射。 初始化：防火墙和selinux关闭，虚拟机的网卡设置成only_host(VirualBox的设置file>Host NetWork Manager添加网卡，dhcp关闭，) 

systemctl stop NetworkManager ###关闭网卡守护进程防止屏蔽子网卡
 
LVS服务器：
网卡ifcfg-enp0s3地址是 10.10.10.5/24
网卡ifcfg-enp0s3:0地址是 10.10.10.100/24
yum install ipvsadm    ###安装ipvsadm集群管理软件
modprobe ip_vs    ###安装出错解决
ipvsadm -Ln
ipvsadm -A -t 10.10.10.100:80 -s rr ###添加集群
ipvsadm -a -t 10.10.10.100:80 -r 10.10.10.101:80 -g ###添加节点
ipvsadm -a -t 10.10.10.100:80 -r 10.10.10.102:80 -g ###添加节点
 
真实服务器：
systemctl stop NetworkManager
yum install nginx
网卡ifcfg-enp0s3地址是 10.10.10.101/24
网卡ifcfg-lo:0地址是 10.10.10.100/32    ###NETMASK 255.255.255.255
 
vim /etc/sysctl.conf    ###编辑内核参数
net.ipv4.conf.all.arp_ignore = 1
net.ipv4.conf.all.arp_announce = 2
net.ipv4.conf.default.arp_ignore = 1
net.ipv4.conf.default.arp_announce = 2
net.ipv4.conf.lo.arp_ignore = 1
net.ipv4.conf.lo.arp_announce = 2
net.ipv4.conf.enp0s3.arp_ignore = 1
net.ipv4.conf.enp0s3.arp_announce = 2
net.ipv4.conf.virbr0.arp_ignore = 1
net.ipv4.conf.virbr0.arp_announce = 2
net.ipv4.conf.virbr0-nic.arp_ignore = 1
net.ipv4.conf.virbr0-nic.arp_announce = 2
sysctl -p   ###刷新
 
route add -host 10.10.10.100 dev lo:0
